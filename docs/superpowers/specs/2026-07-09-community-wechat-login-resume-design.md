# 社区分享微信登录续传设计

## 目标

Android 用户点击“分享到 VD 社区”时，如果尚未登录微信或微信会话已经失效，应用直接发起微信登录。登录成功后回到原文章，并自动继续完成刚才的社区分享。

## 方案

新增一个持久化的一次性待分享任务，保存：

- 录音文件名 `audioName`
- 可选回复目标 `replyToShareId`
- 创建时间

任务存入应用私有 `SharedPreferences`，因此微信授权期间即使 Activity 或应用进程被系统回收，也可以恢复。

## 数据流

1. 文章详情页请求社区分享。
2. 后端返回 `needs_wechat_signin`，或返回 `401 unauthorized`。
3. 应用保存待分享任务；若会话失效，先清除旧 session。
4. 应用调用微信 SDK 发起登录。
5. 微信回调成功，后端交换 code 并保存新 session。
6. 回调 Activity 读取待分享任务，打开对应的 `RecordingDetailActivity`。
7. 文章详情加载完成后原子消费待分享任务，并重新调用社区分享。
8. 分享成功或不可重试失败后，任务保持已删除状态，避免重复发布。

## 错误处理

- 未安装微信或微信 SDK 无法启动：清除待分享任务并提示用户。
- 用户取消或微信授权失败：清除待分享任务，不发布内容。
- code 交换失败：清除待分享任务，显示登录失败原因。
- 登录成功但文章已不存在：消费任务并显示真实社区错误。
- 重复微信回调：只有第一次能消费任务，不会重复分享。

## 导航

- 普通账号页微信登录仍返回账号页，行为不变。
- 只有存在待社区分享任务时，微信登录成功才打开对应文章详情页。
- 详情页通过 `audioName` 恢复文章，不依赖旧 Activity 实例仍然存活。

## 测试

- 待分享任务保存、读取、过期与一次性消费。
- 未登录社区分享会保存任务并调用微信登录。
- 无法启动微信时清除任务。
- 微信登录成功后路由到原文章。
- 登录取消或失败时不路由、不发布并清除任务。
- 详情页只自动续传一次。
- 完整单测、release 构建和真机覆盖安装。
