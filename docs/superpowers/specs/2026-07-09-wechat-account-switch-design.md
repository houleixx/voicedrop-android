# 微信账号切换设计

## 目标

Android 微信登录成功后，将所有 API 请求切换到微信 session 对应的数据空间。退出微信登录后，恢复本地保存的匿名 token 和匿名数据空间。

社区分享触发的登录不再自动重试登录前的匿名文章；登录成功并切换账号后，由用户在微信账号的数据中重新选择文章并点击分享。

## 身份规则

- 匿名 token 始终保存在本地，不因微信登录或退出而删除。
- 未登录时，默认 bearer 为匿名 token。
- 微信登录后，默认 bearer 为微信 session。
- 退出微信后，删除 session，默认 bearer 自动恢复为匿名 token。
- 微信授权 code 交换始终携带匿名 token，不能因为已有 session 而改用 session。

## 账号冲突确认

后端微信登录响应包含 `scope`。客户端将其与当前匿名 token 推导出的 `users/<anonId>/` 比较：

- scope 相同：直接保存 session 并切换。
- scope 不同：提示“该微信已关联另一个云端空间”。说明切换后当前匿名数据暂时不可见，但不会删除，退出微信后可以恢复。

冲突提示按钮：

- `切换到微信账号`：保存 session，切换账号。
- `保留当前账号`：不保存 session，继续使用匿名账号。

## 入口

### 社区分享

1. 匿名用户点击社区分享。
2. 保存登录来源标记并调起微信。
3. 登录成功或确认切换后，清除待分享状态。
4. 打开录音列表并刷新微信账号数据。
5. 提示“已切换到微信账号，请重新选择文章分享”。
6. 不自动分享登录前的匿名文章。

### 设置页

1. 用户点击微信登录。
2. 使用相同的 code 交换、scope 比较和冲突确认。
3. 登录成功后打开录音列表并刷新微信账号数据。

### 退出登录

1. 删除微信 session。
2. 保留匿名 token。
3. 打开录音列表并刷新匿名账号数据。

## 错误处理

- 微信未安装、授权取消或 code 交换失败：不切换账号，清除社区登录来源标记。
- 用户拒绝账号冲突切换：不保存 session，清除社区登录来源标记。
- 已取得但未被用户确认的 session 不写入本地。

## 测试

- bearer 在未登录、登录和退出后的选择。
- 微信 code 交换使用匿名 token。
- scope 相同直接切换，scope 不同要求确认。
- 社区登录成功后不自动重试旧文章。
- 设置入口与社区入口使用同一切换逻辑。
- 退出登录恢复匿名 bearer。
- 完整单测、release 构建和真机覆盖安装。
