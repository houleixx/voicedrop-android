# 图片风格菜单语义图标设计

## 目标

语音详情页的图片风格子菜单不再为所有操作统一显示纸飞机，而是根据每项风格语义显示不同图标。

## 图标映射

| 菜单 ID | 文案 | 图标语义 |
|---|---|---|
| `cartoon` | 卡通 | 调色盘 |
| `ad` | 广告 | 扩音器 |
| `watercolor` | 水彩 | 水滴 |
| `sketch` | 素描 | 铅笔 |
| `oil` | 油画 | 画笔 |
| `film` | 胶片 | 胶片/相机 |

## 实现方式

- 新增六个单色 Android Vector Drawable，使用当前菜单图标相同的视口、描边风格和透明背景。
- 在 Android 客户端按 `UIConfigStore.MenuNode.id` 选择图标，不依赖可本地化的 `label`。
- 现有子菜单右箭头与返回箭头保持不变。
- 未识别、远程新增或缺少 ID 的可执行菜单项继续回退到纸飞机，保证兼容旧配置和未来配置。
- 图标继续由 `menuRow` 使用当前红色着色，尺寸、行高和右侧位置不变。

## 测试与验证

- 源码契约测试验证六个 ID 分别映射到六个不同的图标常量，并验证未知 ID 回退纸飞机。
- Android 资源编译验证六个 Vector Drawable 合法。
- 运行 `./gradlew testDebugUnitTest`。
- 真机或模拟器检查图标在菜单右侧清晰、尺寸统一且没有裁切。

## 跨端影响

本次只修改 Android 表现层，不改变菜单 JSON、指令内容、API 或图片 marker。iOS 与小程序无需同步；若未来希望远程配置任意图标，再单独扩展跨端菜单 schema。
