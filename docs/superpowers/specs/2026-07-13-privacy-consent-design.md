# VoiceDrop Android 首次启动隐私同意设计

## 目标

在 VoiceDrop Android 首次启动时展示隐私保护提示。用户明确同意当前版本的隐私政策前，只运行展示提示所需的本地 UI 初始化，不启动业务逻辑、联网行为、第三方统计或剪贴板邀请归因。

隐私政策地址固定为：`https://voicedrop.cn/privacy/`。

## 用户体验

首次打开 App 时显示不可通过返回键、点击遮罩或滑动关闭的模态弹窗。弹窗包含：

- 标题“隐私保护提示”。
- 简短说明 VoiceDrop 为录音、转写、文章生成和可选分享功能处理用户主动提供的数据。
- 可点击的《隐私政策》链接，使用系统浏览器打开 `https://voicedrop.cn/privacy/`。
- 主按钮“同意并继续”。
- 次按钮“不同意并退出”。

用户选择“同意并继续”后，App 保存本机同意记录、关闭弹窗，并启动正常业务。用户选择“不同意并退出”后关闭当前任务，不启动业务逻辑。

弹窗仅在当前隐私政策版本尚未同意时出现。清除 App 数据、卸载重装、换设备或代码中的当前政策版本升级后，需要重新同意。

## 同意状态

同意记录保存在 App 私有的 `SharedPreferences`，不上传服务器、不依赖账号，并包含：

- 当前已同意的隐私政策版本：`2026-07-12`。
- 同意时间戳。

是否允许启动业务只由“本机保存的已同意版本是否等于代码中的当前版本”决定。布尔字段不是必要的独立真相来源，避免它与版本字段不一致。

## 启动门禁

`Application.onCreate()` 在未同意时只允许完成 DialogX 样式及 Activity 外观回调等本地 UI 基础配置。以下行为必须延迟：

- 友盟 `preInit`、`init` 和自动页面统计配置。
- `ReferralManager.runOnLaunch()`，包括首次剪贴板邀请码读取和归因请求。
- 匿名身份创建或账号相关初始化。
- 录音列表刷新、HTTP 请求、WebSocket 连接、更新检查及其他业务后台任务。

首次同意后，由一个幂等的应用级入口激活友盟和邀请归因。已经同意的用户后续冷启动时，`Application` 可以直接调用同一入口。入口必须防止同一进程重复初始化。

`RecordingsActivity` 在创建任何业务对象或启动页面业务前执行门禁检查。未同意时只建立承载隐私弹窗所需的最小界面；同意回调再继续执行原有首页初始化。由于首页同时承接启动器和 VoiceDrop 深链，这两种入口都受同一门禁保护。

`ShareCollectActivity` 是独立导出的系统分享入口，也必须执行同一门禁；未同意时先征求同意，同意后继续原始分享接收流程，不同意则退出。

微信回调入口只能由用户在已进入正常业务后主动发起，本次不增加独立弹窗；其回调仍依赖此前已经保存的同意状态。

## 组件边界

- 新增纯状态组件：定义政策 URL、当前政策版本，并读写本地同意版本和时间。
- 新增隐私弹窗组件：只负责文案、链接、不可取消行为以及同意/退出回调。
- `VoiceDropApplication`：保留 UI 基础初始化，提供幂等的“激活已同意服务”入口。
- `RecordingsActivity` 与 `ShareCollectActivity`：在现有初始化前执行门禁，同意后调用原有初始化流程。
- `AboutActivity`：“隐私说明”改为打开正式隐私政策网页，保证用户以后可以随时访问。

## README 与政策一致性

删除 README 中“友盟统计”章节，不移除友盟依赖、构建参数、发布密钥检查或测试。

线上隐私政策目前写有“没有第三方分析或追踪”，与 Android 友盟统计实际行为不一致。本次按用户要求不修改线上政策，也不删除友盟；该政策内容必须在正式上架前另行修正。首次启动弹窗不能消除政策披露不准确的问题。

## 错误与边界处理

- 隐私政策网页无法打开时保留弹窗并给出简短提示，不把打不开网页视为同意。
- Activity 重建时不得叠加多个弹窗或重复启动业务。
- 同意写入本地后再激活业务，确保进程异常结束时不会出现“业务已运行但同意未保存”。
- 应用级业务激活入口必须幂等，避免 Activity 重建导致友盟或邀请归因重复初始化。
- Android 麦克风和相机运行时权限仍在用户实际使用对应功能时申请，不并入首次隐私弹窗。

## 测试与验收

采用 JVM 单元测试和源代码约束测试覆盖：

- 当前政策版本未同意时门禁拒绝启动业务。
- 同意当前版本后门禁放行，并保存版本和时间。
- 旧版本同意记录不会放行新版本。
- `VoiceDropApplication.onCreate()` 不再无条件初始化友盟或执行邀请归因。
- 首页和系统分享入口都在业务初始化前执行隐私门禁。
- 弹窗包含正确 URL、“同意并继续”和“不同意并退出”，且不可取消。
- README 不再包含友盟统计章节。

完成后运行 `./gradlew testDebugUnitTest`；构建验证运行 `./gradlew assembleDebug`。弹窗展示、返回键/遮罩不可关闭、网页跳转、同意后业务启动以及不同意退出还需要模拟器或真机手动验证。

## 跨端影响

本次只修改 Android 仓库，不修改 iOS、小程序或后端。隐私政策版本常量仅控制 Android 本机是否重新征求同意，不改变任何 HTTP、WebSocket、文章 schema、录音命名或图片 marker 契约。
